### PostCompact Hook — CRITICAL CORRECTION
PostCompact does **NOT** receive `compacted_content` on stdin. It receives only: `{session_id, transcript_path, cwd, hook_event_name}`. Open GitHub issues #14258 and #40492 request this — NOT YET IMPLEMENTED as of 2026-05-26. Correct PostCompact use: unconditional re-injection via `head -30 AGENTS.md` OR read `transcript_path` to check content survival.

### PreCompact Hook — CAN BLOCK
`PreCompact` fires BEFORE compaction and CAN block (exit 2 or `decision: "block"`). It has a `matcher` field: `"manual"`, `"auto"`, or `"*"`. Scaffold templates use `PostCompact` for context re-injection (after compaction), not `PreCompact`.

### Stop Hook — Decision Pattern
Two mechanisms: (1) `exit 2` — silently blocks stop. (2) `{"decision":"block","reason":"..."}` on stdout + exit 0 — blocks stop AND feeds reason to Claude. For test-enforcement, exit 2 is sufficient.
**v2.1.143**: Stop hooks that block repeatedly now end the turn with a warning after 8 consecutive blocks; override with `CLAUDE_CODE_STOP_HOOK_BLOCK_CAP`.

### Skill Scoping Priority — CORRECTED
Official order (Managed > CLI flag > Project > User > Plugin):
- **Project** skills (`.claude/skills/`) override **User** skills (`~/.claude/skills/`) when names collide
- Previous documentation saying "Personal overrides project" was WRONG

### Skills Progressive Disclosure
3 stages: (1) Discovery — name + description only at startup; (2) Activation — full SKILL.md loaded when task matches; (3) Execution — supporting files loaded on demand. Subagents can use `skills:` frontmatter to preload content.

## Libraries

### Zod
Current stable: v4.x. **CRITICAL CORRECTION from cache 2026-05-08:**
- Error customization param is **`error`** (not `message`). `message` is still supported but deprecated in v4. Correct: `z.email({ error: "Invalid email address" })`.
- Error formatting: `z.flattenError(error)` available (replaces `.flatten()` on ZodError instances); `z.treeifyError(error)` for nested tree; `z.prettifyError(error)` for human-readable string.
- `z.formatError()` deprecated — use `z.treeifyError()`.
- Chained string-format methods deprecated: `z.string().url()` → `z.url()`, `z.string().datetime()` → `z.iso.datetime()`, `z.string().email()` → `z.email()`, `z.string().uuid()` → `z.uuid()`.
- `z.uuid()` stricter (RFC 9562/4122); `z.guid()` for permissive UUID-like validation.
- `z.string().ip()` → `z.ipv4()` / `z.ipv6()`; `z.string().cidr()` → `z.cidrv4()` / `z.cidrv6()`.
- `.merge()` on ZodObject deprecated → use `.extend()`.
- `z.interface()` new API for object types.
